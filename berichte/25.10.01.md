# Grundlagen

Shell: Ist ein Programm das eine textorientierte Schnittstelle zum Betriebssystem zur Verfügung stellt.

Prompt:

```sh
~/workspace$
```

Einfacher Befehl:
```sh
$ date
```

Jeder Befehl hat Optionen über die das Verhalten gesteuert werden kann.

```sh
$date -I
```

## Optionen
- short-hand = 1 Buchstabe
- long-hand  = Wort

```sh
$date --iso-8601
```

Quellen zu Informationen über einen Befehl:

- häufig: Option `--help`
- man pages (Manuel = Handbuch) `$ man date`
Nicht in Replit aber in gängigen Linux Distributionen
- Internetsuche mit "man date"
- KI

## Argumente

Daten mit denen ein Befehl arbeitet.

```sh
$ echo Hallo Welt
```
`Hallo Welt` ist hier das Argument

mit Optionen

```sh
$ echo -n Hallo Welt
```

## Tastenkürzel

- Pfeil nach oben/unten: Hisorie
- Tabulator: automatische File Vervollständigung
- Strg-A: Anfang der Zeile
- Strg-C: Abbruch des aktuellen Kommandos

## File Befehle

cd, mkdir, ls, touch, pwd, rm, rmdir

- working directory (Arbeitsverzeichnis)

- Unterschied absoluter und relativer Pfad

- Bedeutung von cd (ohne Argument) und von ~


### cd
```sh
$ cd
```
- `cd` ist ein Befehl
- `cd` ist die Abkürzung von "change directory" (Verzeichnis ändern)
- Man benutzt es wenn man in ein anderes Arbeitsverzichnis gehen möchte

### mkdir
```sh
$ mkdir
```
- `mkdir` ist ein Befehl
- `mkdir` ist die Abkürzung von "make directory" (Verzeichnis erstellen)
- Man verwendet es um ein neues Verzeichnis (Ordner) zu erstellen

### ls
```sh
$ ls
```
- `ls` ist ein Befehl
- `ls` ist die Ablürzung von "list" (Liste)
- Man verwendet es um den Inhalt des Verzeichnis zu zeigen

### touch
```sh
$ touch
```
- `touch` ist ein Befehl
- `touch` ist kein Abkürzung
- man verwendet es im primären
